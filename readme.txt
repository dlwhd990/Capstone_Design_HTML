각 팀원이 작업한 파일들의 디렉토리와 해당 파일에 대한 설명입니다.

1. 이종혁

(1) editor.html (최상위 디렉토리)
 - 에디터 페이지의 구조를 구성한 HTML 파일입니다.

(2) index.html (최상위 디렉토리)
 - 웹사이트의 메인 화면 구조를 구성한 HTML 파일입니다.

(3) login.html (최상위 디렉토리)
 - 웹사이트의 로그인을 담당하는 페이지의 구조를 구성한 HTML 파일입니다.

(4) signup.html (최상위 디렉토리)
 - 웹사이트의 회원가입을 담당하는 페이지의 구조를 구성한 HTML 파일입니다.

(5) 모든 css 파일 (/css)
 - 웹사이트의 모든 페이지들을 구성한 HTML 파일을 꾸며주는 stylesheet 역할을 하는 파일입니다.

(6) 모든 js 파일 (/src)
 - 웹사이트의 모든 페이지들을 동적인 페이지로 만들어주는 자바스크립트 파일들입니다.
  예를 들어 에디터 페이지의 이미지 편집 기능이나 소스 이미지 검색, 추천과 같은 기능들은
  editor.js 파일에 의해 작동되게 되는 등 사용자와의 상호작용이 가능하도록 만들어주는 역할을
  하는 파일들입니다.

(7) 메인 페이지의 이미지들 (/images)
 - 웹사이트의 메인 페이지를 꾸며주는 이미지들입니다.

2. 최성훈

(1) editor_mobile.html (최상위 디렉토리)
 - 이미지 에디터 페이지를 모바일로 접속했을 시에 모바일 디바이스의 화면 크기에 
    적절하게 출력될 수 있도록 페이지의 구조를 구성한 HTML 파일입니다.

(2) editor_mobile.css (/css)
 - 에디터 페이지의 모바일 버전 HTML 파일을 꾸며주는 stylesheet 역할을 하는 파일입니다.
   모바일 버전에 맞게 작은 화면에서도 효율적으로 사용가능하도록 설계하였습니다.

(3) editor_mobile.js (/src)
 - 에디터 페이지의 모바일 버전 페이지를 사용자와의 상호작용이 가능한 동적인 페이지로 만들어주는
  자바스크립트 파일입니다. 모바일 버전의 제한된 환경에서도 원활하게 이미지 편집이 가능하도록
  구현하였습니다. 

(4) editor.js (/src)
 - 에디터 페이지의 PC버전의 기능 몇가지를 수정하고 추가하는 작업을 하였습니다. 유저의 편의성을
  최우선적으로 생각하여 몇 번의 토의를 통해 페이지를 나가기 전에 확인 과정을 거치거나
 캔버스 상에 올라가는 이미지의 크기에 대한 내용 등에 대한 수정, 추가 작업을 수행하였습니다.

3. 양혜강

(1) help.html (최상위 디렉토리)
 - 웹사이트에 대해 설명을 해주어 사용자에게 도움을 줄 수 있는 페이지의 구조를 구성한 HTML 파일입니다.

(2) help.css (/css)
 - 도움말 페이지를 구성한 help.html 파일을 꾸며주는 stylesheet 역할을 하는 파일입니다.

(3) editor.css (/css)
 - 에디터 페이지를 구성한 editor.html 파일을 꾸며주는 stylesheet 역할을 하는 파일입니다. 이 부분에서
  사용자의 편의성을 고려한 부분을 디자인 하였습니다.

(4) index.css (/css)
 - 웹사이트의 메인 페이지를 꾸며주는 stylesheet 역할을 하는 파일입니다. 이 부분에서 더 친숙한 UI를
   구성하기 위해 수정작업을 하였습니다.

(5) help 이미지들 (/images)
 - 도움말 페이지를 꾸며주는 이미지들을 직접 제작하여 추가하였습니다.

4. 김한준

(1) login.js (/src)
 - 웹사이트에서 사용자가 로그인을 하면 Firebase상에서 auth 과정을 거쳐 로그인이 되고 세션에
  현재 로그인 된 사용자의 정보가 들어가서 로그인 상태를 유지해주는 역할을 하는 파일입니다.

(2) signup.js (/src)
 - 웹사이트에서 사용자가 회원가입을 하면 기존의 회원들 중 중복되는 아이디가 있는지, 아이디는
  이메일 형식에 맞는지, 비밀번호가 6자리 이상인지 등의 조건을 만족하는 지 확인 한 후에
   걸리는 조건이 없다면 성공적으로 회원가입을 시켜주는 역할을 하는 파일입니다. 

(3) loginCheck.js (/src)
 - 모든 페이지의 HTML 파일들과 연결되어 모든 페이지에서 현재의 로그인 상태를 확인할 수 있도록
   해주는 역할을 하는 파일입니다. 이 파일을 활용하여 로그인이 되어있는 사용자만을 위한 기능을
   제공해주어야할지 말아야할지 판단하거나 헤더 부분에 로그인/로그아웃 버튼 중 어떤 것이 활성화
   되고 보여져야 할지 판단할 수 있는 정보를 제공해주는 역할도 합니다.

(4) 소스 이미지들 (외부 데이터베이스에 있습니다)
 - 웹사이트에서 제공하는 소스 이미지들을 제공하였습니다.

5. 이정욱

(1) 소스 이미지들 (외부 데이터베이스에 있습니다)
 - 웹사이트에서 제공하는 소스 이미지들을 제공하였습니다.
